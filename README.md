# Olist Brazilian E-Commerce: SQL Analysis

Аналитический проект на реальном датасете бразильского маркетплейса Olist.

## Описание проекта

Проект выполнен в рамках самостоятельного изучения SQL и подготовки к позиции Junior Data Analyst / Data Scientist. Все запросы написаны на диалекте **SQLite**.

Датасет: [Brazilian E-Commerce Public Dataset by Olist](https://www.kaggle.com/datasets/olistbr/brazilian-ecommerce)

## Решенные аналитические задачи

| № | Задача | Используемые техники |
|---|--------|---------------------|
| 1 | Помесячная динамика заказов | `STRFTIME`, `GROUP BY` |
| 2 | Топ-10 категорий по выручке | `JOIN`, `GROUP BY`, `ORDER BY`, `LIMIT` |
| 3 | Города с наибольшей долей оценок "5" | `JOIN`, `WHERE`, `GROUP BY` |
| 4 | Среднее время доставки по штатам | `JULIANDAY`, `AVG`, `GROUP BY` |
| 5 | Клиенты без повторных покупок | `GROUP BY`, `HAVING COUNT` |
| 6 | Самые быстрые доставки | `JULIANDAY`, `JOIN`, `ORDER BY` |
| 7 | Товары, которые никогда не оплачивали картой | `NOT IN`, подзапрос |
| 8 | Средняя стоимость заказа по штатам продавцов | Оконная функция `AVG() OVER(PARTITION BY)` |
| 9 | Рейтинг продавцов по выручке внутри штата | `ROW_NUMBER() OVER(PARTITION BY ... ORDER BY)` |
| 10 | Скользящее среднее выручки продавцов (3 месяца) | `AVG() OVER(... ROWS BETWEEN 2 PRECEDING AND CURRENT ROW)` |

## Используемые технологии

- **SQL** (SQLite)
- **Оконные функции:** `ROW_NUMBER`, `AVG` с `PARTITION BY`, рамки окна (`ROWS BETWEEN`)
- **Агрегация и группировка:** `GROUP BY`, `HAVING`
- **Подзапросы:** `NOT IN`, коррелированные подзапросы
- **Работа с датами:** `JULIANDAY`, `STRFTIME`
